# Programa operativo para setters

## Objetivo

Montar un piloto comercial simple para que setters principiantes agenden reuniones de calidad para landing pages y sitios web.

## Diseno del piloto

Recomendacion para empezar:

- arrancar con `1 o 2 setters`
- correr el piloto durante `2 semanas`
- revisar llamadas y resultados casi a diario
- escalar a `3 o mas` solo cuando ya exista una formula repetible

La prioridad es validar conversion, no volumen bruto.

## Modelo operativo

- el setter agenda
- el closer vende
- el brief conecta ambos roles
- el CRM o tablero guarda el estado real

El setter no debe cargar con cierre, propuesta ni negociacion final.

## Onboarding recomendado

### Dia 1: producto

- diferencia entre landing y sitio web
- problemas que resuelve cada uno
- glosario tecnico minimo
- promesas que si y no se pueden hacer

### Dia 2: ventas

- apertura
- diagnostico
- lenguaje comercial
- manejo de objeciones
- cierre a reunion

### Dia 3: practica

- roleplays
- correccion en vivo
- primeras llamadas supervisadas

## Practica supervisada de las primeras 2 semanas

Revisa varias llamadas por persona con foco en:

- tono y claridad
- apertura
- capacidad de detectar dolor
- uso de objeciones
- cierre a reunion
- calidad del brief

## Roleplays obligatorios

Cada setter debe pasar al menos estos escenarios:

- `precio`
- `ya tengo web`
- `no tengo tiempo`
- `mandame WhatsApp`
- `llamame luego`

## Scorecard de evaluacion

Califica cada llamada o roleplay de 1 a 5 en:

- apertura
- diagnostico
- claridad de oferta
- manejo de objeciones
- cierre a reunion
- calidad del handoff

Un setter puede pasar a mas autonomia cuando mantenga `4/5` o mas en la mayoria de categorias.

## Politica de comision recomendada

### Regla base

Pagar `10% del primer pago efectivamente cobrado` del proyecto base.

### Condiciones para pagarla

La comision aplica solo si:

- la reunion fue agendada por ese setter
- el handoff quedo registrado de forma completa
- la reunion fue asistida o al menos entregada correctamente al closer
- la venta se cerro dentro de una ventana de `30 dias` desde la reunion
- el cliente ya hizo el primer pago

### Lo que no paga comision en esta fase

- upsells futuros
- mensualidades de mantenimiento
- renovaciones
- ventas cerradas fuera de la ventana definida
- leads que ya estaban negociandose antes del setter

### Forma recomendada de liquidar

Para mantenerlo simple al inicio:

- corte una vez por semana o una vez por mes
- pagar solo negocios cobrados
- registrar el monto base del primer pago y la comision resultante

## Ejemplo simple

Si el primer pago cobrado del proyecto fue `1,000`, la comision del setter seria `100`.

Si el cliente paga despues una mensualidad o un upsell, eso no entra en esta fase.

## Reglas de atribucion

Para evitar discusiones:

- el setter duenio es quien deja primero la reunion agendada y el brief completo
- si otro miembro del equipo interviene despues, no cambia la atribucion inicial
- si el closer detecta brief flojo o informacion falsa, puede marcar revision antes de aprobar la comision

## Senales minimas para avanzar a escala

No sumes mas setters hasta ver algo de esto:

- primeras `5 ventas` cerradas
- show rate consistente
- brief de buena calidad
- objeciones bien manejadas
- una apertura que el equipo ya puede repetir

## Riesgos a vigilar

- contratar demasiada gente antes de validar el discurso
- llenar al equipo de servicios fuera de foco
- medir solo llamadas hechas y no reuniones asistidas
- pagar comisiones sin reglas claras de atribucion

## Criterio de exito del sistema

El sistema va bien cuando:

- el setter entiende que vende una reunion, no un proyecto entero
- el closer recibe contexto util
- el lead llega mejor preparado
- la tasa de cierre mejora con el tiempo

## Siguiente mejora cuando el piloto funcione

Despues de validar el piloto humano, puedes considerar:

- bono pequeno por reunion asistida
- mas setters
- mas canales de prospeccion
- automatizar partes del seguimiento y confirmacion
